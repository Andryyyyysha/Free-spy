# Free Spy

Бесплатный инструмент с открытым исходным кодом для логирования измененных и удаленных сообщений в Telegram через меню «Автоматизация чатов». 

Проект разработан на базе библиотеки aiogram 3 и предназначен для развертывания личного экземпляра бота.

[![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com/deploy?repo=https://github.com/Claxy-mod/Free-spy)

---

## Варианты установки

Вы можете запустить собственного бота двумя способами:
1. **Автоматическая установка в 1 клик** (самый простой и быстрый способ).
2. **Ручная установка** (дает полный контроль над базой данных).

---

## 🚀 Вариант 1. Автоматическая установка в 1 клик

Нажмите на кнопку выше («Deploy to Render») и следуйте инструкциям:
1. Войдите в свой аккаунт на **Render** (или зарегистрируйтесь).
2. В появившейся форме введите:
   - `BOT_TOKEN` — токен вашего бота от @BotFather.
   - `USER_ID` — ваш числовой ID в Telegram (резервный адрес).
3. Render автоматически создаст бесплатную базу данных PostgreSQL, сгенерирует надежный ключ шифрования `ENCRYPTION_KEY` и запустит вашего бота!
4. После успешного деплоя перейдите к **Шагу 4** и **Шагу 5** ручной инструкции для настройки непрерывной работы (UptimeRobot) и подключения бота в Telegram.

> [!NOTE]
> База данных PostgreSQL на бесплатном тарифе Render предоставляется бесплатно на 90 дней. По истечении этого срока базу нужно будет пересоздать или перейти на платный тариф. Для постоянного бесплатного использования рекомендуется использовать **Вариант 2** с базой данных от Supabase.

## 🛠 Вариант 2. Ручная установка (Render + Supabase)

### Шаг 1. Создание базы данных Supabase

1. Зарегистрируйтесь на сайте Supabase.
2. Создайте новый проект и задайте пароль для базы данных.
3. В правом верхнем углу панели управления проектом нажмите Connect.
4. Выберите вкладку Connection String -> URI, измените метод подключения на Transaction Pooler.
5. Скопируйте полученную строку подключения. Она имеет следующий вид:
   `postgresql://postgres.xxxx:[YOUR-PASSWORD]@aws-0-xxxx.pooler.supabase.com:6543/postgres`
6. Замените плейсхолдер `[YOUR-PASSWORD]` на ваш пароль от базы данных. Если пароль содержит специальные символы, закодируйте их (например, замените `,` на `%2C`, а `/` на `%2F`).

---

### Шаг 2. Получение ключа шифрования

Для работы шифрования необходимо сгенерировать секретный ключ. Запустите в терминале скрипт:
```bash
python generate_key.py
```
Сгенерированная строка понадобится при настройке хостинга.

---

### Шаг 3. Деплой приложения на Render

1. Скопируйте данный репозиторий в свой аккаунт на GitHub.
2. Зарегистрируйтесь на Render и создайте новый веб-сервис (New Web Service).
3. Выберите репозиторий с ботом.
4. Укажите следующие параметры конфигурации:
   * Language: `Python`
   * Build Command: `pip install -r requirements.txt`
   * Start Command: `python main.py`
   * Instance Type: `Free`
5. В меню Advanced добавьте переменные окружения (Environment Variables):
   * `BOT_TOKEN` — токен вашего Telegram-бота, полученный от @BotFather.
   * `USER_ID` — ваш числовой ID в Telegram (используется как резервный адрес).
   * `DATABASE_URL` — строка подключения к Supabase, полученная на Шаге 1.
   * `ENCRYPTION_KEY` — ключ шифрования, сгенерированный на Шаге 2.
   * `TIMEZONE_NAME` — ваше локальное системное время (например, `UTC`, `Europe/Moscow`, `Asia/Tashkent` и так далее).
6. Нажмите Create Web Service и дождитесь окончания сборки.

---

### Шаг 4. Настройка непрерывной работы (UptimeRobot)

Бесплатный тариф Render приостанавливает работу сервиса при отсутствии входящих запросов в течение 15 минут. Чтобы бот работал непрерывно:
1. Скопируйте адрес вашего веб-сервиса (выглядит как `https://your-app.onrender.com`).
2. Создайте бесплатный монитор на UptimeRobot с типом HTTPS.
3. Укажите скопированный URL и интервал проверки 5 минут. Встроенный в бота веб-сервер будет отвечать на запросы монитора, предотвращая отключение хостинга.

---

### Шаг 5. Подключение бота в Telegram

1. Откройте настройки вашего профиля в Telegram.
2. Перейдите в раздел «Автоматизация чатов» (или «Чат-боты»).
3. Нажмите «Добавить бота» и укажите имя вашего бота.
4. Задайте права доступа для чатов.
5. Если бот уже был привязан ранее, переподключите его (отключите и включите заново), чтобы данные обновились в новой базе данных.

---

## Локальный запуск

1. Склонируйте репозиторий и установите зависимости:
```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```
2. Создайте файл конфигурации `config.ini` на основе шаблона:
```bash
cp config.ini.example config.ini
```
3. Заполните переменные в `config.ini` своими данными.
4. Запустите приложение:
```bash
python main.py
```
