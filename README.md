# Free Spy: Бесплатная OpenSource замена Dialog Spy Bot

**Free Spy** — это современный Telegram-бот на базе библиотеки `aiogram 3`, предназначенный для перехвата, сохранения и логирования удаленных и измененных сообщений в ваших бизнес-чатах Telegram.

## Возможности
* 📁 **Поддержка любых типов сообщений**: бот логирует не только текст, но и фото, видео, голосовые сообщения (ГС), видео-сообщения (кружки), файлы, аудио, стикеры и GIF.
* 🗑 **Отслеживание удалений и изменений**: при удалении или изменении сообщения бот мгновенно отправляет уведомление владельцу с указанием автора.
* 👥 **Мультипользовательская архитектура**: бот автоматически перенаправляет уведомления именно тому пользователю, который подключил его к своему бизнес-аккаунту.
* 🔒 **Абсолютная приватность и шифрование (Fernet AES)**: Тексты всех сообщений шифруются «на лету» перед отправкой в базу данных. Владелец хостинга или базы данных (даже при утечке БД) увидит в таблицах только зашифрованную информацию. Расшифровать её может только сам запущенный бот с помощью секретного ключа.

## Полное руководство по запуску своего бота на Render (24/7 Бесплатно)

Для работы бота в облаке 24/7 без затрат мы будем использовать связку **Render.com** (бесплатный хостинг), **Supabase** (бесплатная облачная база данных PostgreSQL) и **UptimeRobot** (поддержание активности).

### Шаг 1. Создание облачной базы данных в Supabase
1. Перейдите на [Supabase.com](https://supabase.com/) и создайте бесплатный аккаунт.
2. Создайте новый проект (Project). Придумайте имя и пароль для базы данных (обязательно сохраните его!).
3. В шапке проекта (в правом верхнем углу) нажмите кнопку **Connect**.
4. В открывшейся вкладке выберите **Connection String** -> **URI** и переключите *Connection Method* на **Transaction Pooler**.
5. Скопируйте полученную строку вида:
   `postgresql://postgres.xxx:[YOUR-PASSWORD]@aws-0-xxx.pooler.supabase.com:6543/postgres`
6. Замените `[YOUR-PASSWORD]` на ваш реальный пароль от базы данных. 

> ⚠️ **Внимание:** Если в вашем пароле есть спецсимволы (например, запятая `,` или слэш `/`), их необходимо закодировать в URL-формат. Например, `,` заменяется на `%2C`, а `/` на `%2F`.

---

### Шаг 2. Генерация ключа шифрования
Для безопасности переписки запустите локально скрипт `generate_key.py`:
```bash
python generate_key.py
```
Скрипт сгенерирует для вас уникальный ключ Fernet. Сохраните его — мы пропишем его на хостинге под именем `ENCRYPTION_KEY`.

---

### Шаг 3. Деплой бота на Render.com (24/7)
1. Сделайте **Fork** (ответвление) или скопируйте этот репозиторий к себе в **приватный** или **публичный** репозиторий на GitHub.
2. Зарегистрируйтесь на [Render.com](https://render.com) (карту привязывать не нужно).
3. Создайте новый сервис: **New -> Web Service**. Выберите **Build and deploy from a Git repository** и подключите ваш репозиторий.
4. Укажите следующие настройки:
   * **Language**: `Python`
   * **Build Command**: `pip install -r requirements.txt`
   * **Start Command**: `python main.py`
   * **Instance Type**: `Free` (0$ / month)
5. Нажмите кнопку **Advanced** и в разделе **Environment Variables** добавьте 5 переменных окружения:
   * `BOT_TOKEN` — токен вашего бота от `@BotFather`.
   * `USER_ID` — ваш личный Telegram ID.
   * `TIMEZONE_NAME` — `Europe/Moscow`.
   * `DATABASE_URL` — строка подключения к Supabase, которую мы скопировали на Шаге 1.
   * `ENCRYPTION_KEY` — ключ шифрования, сгенерированный на Шаге 2.
6. Нажмите **Create Web Service**. Бот соберется и запустится за пару минут.

---

### Шаг 4. Настройка UptimeRobot (чтобы бот не спал)
Поскольку Render усыпляет бесплатные веб-сервисы через 15 минут простоя, мы настроим постоянный пинг:
1. Скопируйте ссылку на ваше приложение Render в левом верхнем углу (она выглядит как `https://free-spy-xxx.onrender.com`).
2. Зарегистрируйтесь на бесплатном сервисе [UptimeRobot.com](https://uptimerobot.com/).
3. Нажмите **Add New Monitor**:
   * **Monitor Type**: `HTTPS`
   * **Friendly Name**: `Spy Bot`
   * **URL (or IP)**: вставьте вашу ссылку от Render.
   * **Monitoring Interval**: `Every 5 minutes` (Каждые 5 минут).
4. Сохраните монитор. Наш встроенный веб-сервер будет отвечать UptimeRobot и бот никогда не уснет.

---

### Шаг 5. Подключение бота в Telegram Business
1. Откройте **Telegram** на телефоне и перейдите в ваш **Профиль**.
2. В правом верхнем углу нажмите **Изменить** (Edit).
3. В списке найдите раздел **Автоматизация чатов** (Chat Automation) или **Чат-боты** (Chatbots).
4. Нажмите **Добавить бота** (Add bot) и укажите юзернейм вашего бота.
5. Настройте права доступа (выберите чаты, в которых бот сможет работать) и сохраните.
6. **Если вы уже подключали бота ранее:** выключите его в этом меню и включите заново (это нужно, чтобы бот получил событие подключения и записал ваш ID в новую облачную базу данных).

## Локальная установка и запуск

Если вы хотите запустить бота локально на компьютере:

1. Установите зависимости:
```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```
2. Скопируйте пример конфигурации `config.ini.example` в `config.ini`:
```bash
cp config.ini.example config.ini
```
3. Заполните `config.ini` вашими данными (токен, ID, ключ шифрования).
4. Запустите скрипт:
```bash
python main.py
```
