# Инструкция по установке Free Spy

Вы можете развернуть бота в облаке Render (бесплатно) или запустить его локально.

---

## Вариант 1. Автоматический деплой на Render

Самый быстрый способ запуска с автоматическим созданием базы данных PostgreSQL и генерацией ключей шифрования.

1. Нажмите кнопку деплоя:
   [![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com/deploy?repo=https://github.com/Claxy-mod/Free-spy)
2. Войдите в свой аккаунт на **Render** (или зарегистрируйтесь).
3. Заполните обязательные переменные:
   * `BOT_TOKEN` — токен вашего бота от @BotFather.
   * `USER_ID` — ваш числовой ID в Telegram.
4. Дождитесь завершения сборки и перейдите к [настройке UptimeRobot](#настройка-непрерывной-работы-uptimerobot).

> [!NOTE]
> На бесплатном тарифе база данных PostgreSQL на Render работает 90 дней. По истечении этого срока базу нужно будет пересоздать. Для постоянной работы рекомендуется использовать **Вариант 2** с базой данных от Supabase.

## Вариант 2. Ручная установка (Render + Supabase)

Данный вариант позволяет бесплатно запустить бота для работы в режиме 24/7 с постоянной базой данных.

### Шаг 1. Создание базы данных Supabase

1. Зарегистрируйтесь на [Supabase](https://supabase.com/).
2. Создайте новый проект и задайте пароль для базы данных.
3. In панели управления проектом нажмите **Connect** (в правом верхнем углу).
4. Перейдите во вкладку **Connection String** -> **URI**, выберите метод подключения **Transaction Pooler**.
5. Скопируйте полученную строку подключения. Она выглядит так:
   `postgresql://postgres.xxxx:[YOUR-PASSWORD]@aws-0-xxxx.pooler.supabase.com:6543/postgres`
6. Замените плейсхолдер `[YOUR-PASSWORD]` на ваш пароль от базы данных. Если пароль содержит спецсимволы, закодируйте их (например, `,` заменить на `%2C`, а `/` на `%2F`).

### Шаг 2. Получение ключа шифрования

Сгенерируйте секретный ключ для шифрования сообщений. Для этого запустите в терминале:
```bash
python generate_key.py
```
Скопируйте сгенерированную строку.

### Шаг 3. Деплой приложения на Render

1. Сделайте форк (fork) этого репозитория в свой аккаунт на GitHub.
2. Зарегистрируйтесь на [Render](https://render.com/) и создайте новый веб-сервис (**New Web Service**).
3. Подключите репозиторий с ботом.
4. Задайте настройки:
   * **Language**: `Python`
   * **Build Command**: `pip install -r requirements.txt`
   * **Start Command**: `python main.py`
   * **Instance Type**: `Free`
5. В разделе **Advanced** добавьте переменные окружения (**Environment Variables**):
   * `BOT_TOKEN` — токен бота от @BotFather.
   * `USER_ID` — ваш числовой ID в Telegram.
   * `DATABASE_URL` — строка подключения к Supabase из Шага 1.
   * `ENCRYPTION_KEY` — ключ шифрования из Шага 2.
   * `TIMEZONE_NAME` — ваше локальное время (например, `Europe/Moscow`, `Asia/Tashkent` и т.д.).
6. Нажмите **Create Web Service**.

## Настройка непрерывной работы (UptimeRobot)

Бесплатный тариф Render приостанавливает работу сервиса при отсутствии входящих запросов в течение 15 минут. Чтобы бот работал непрерывно:
1. Скопируйте адрес вашего веб-сервиса (выглядит как `https://your-app.onrender.com`).
2. Создайте бесплатный монитор на [UptimeRobot](https://uptimerobot.com/) с типом **HTTPS**.
3. Укажите скопированный URL и интервал проверки 5 минут. Встроенный в бота веб-сервер будет отвечать на запросы монитора, предотвращая отключение хостинга.

## Подключение бота в Telegram

1. Откройте настройки вашего профиля в Telegram.
2. Перейдите в раздел **«Автоматизация чатов»** (или «Чат-боты»).
3. Нажмите **«Добавить бота»** и укажите юзернейм вашего бота.
4. Задайте права доступа для чатов.
5. Если бот уже был привязан ранее, переподключите его (отключите и включите заново), чтобы обновить соединение.

## Локальный запуск

1. Склонируйте репозиторий и настройте виртуальное окружение:
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   ```
2. Создайте файл конфигурации `config.ini` на основе шаблона:
   ```bash
   cp config.ini.example config.ini
   ```
3. Заполните параметры в `config.ini` своими данными.
4. Запустите бота:
   ```bash
   python main.py
   ```
